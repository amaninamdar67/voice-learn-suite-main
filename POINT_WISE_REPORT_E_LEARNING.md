# INTRODUCTION - E LEARNING USING AI

---

## # WHAT IS E-LEARNING USING AI

The "E-Learning Using AI" platform is a comprehensive, full-stack web application designed to revolutionize educational technology. It combines artificial intelligence, voice accessibility, and real-time analytics to create an immersive, accessible, and data-driven learning environment. The platform addresses critical gaps in traditional learning management systems by integrating cutting-edge web technologies with AI-powered educational features.

---

## # WHY THIS PROJECT EXISTS

- Traditional Learning Management Systems lack personalization and adaptive learning capabilities.
- Limited accessibility features for blind and visually impaired users prevent inclusive education.
- No AI-powered tutoring support to assist students with personalized learning paths.
- Disconnected communication channels between students, teachers, mentors, and parents.
- Absence of real-time analytics to track student progress and engagement effectively.
- No voice navigation support for users with disabilities or accessibility needs.

---

## # WHAT MAKES IT UNIQUE

- "AI Tutor" - DeepSeek integration for document analysis and intelligent question-answering.
- "Voice Navigation" - Hindi and English language support with spacebar activation.
- "Real-Time Analytics" - Live dashboards for Admin, Mentor, and Parent roles.
- "Multi-Role System" - Six distinct user roles with customized experiences.
- "Accessibility First" - WCAG 2.1 Level AA compliance for inclusive design.
- "Community Features" - Anonymous posting and threaded discussions.
- "Gamification" - Leaderboards and performance tracking for engagement.
- "Multi-Tenant Architecture" - Complete data isolation for multiple institutions.

---

## # HOW IT WORKS

- "Frontend" - React 18 with TypeScript, Material-UI, Tailwind CSS, and Vite.
- "Backend" - Node.js with Express.js framework for API management.
- "Database" - PostgreSQL via Supabase with Row-Level Security policies.
- "AI Engine" - Ollama with DeepSeek-R1 model for local AI inference.
- "Authentication" - Supabase Auth with JWT tokens for secure access.
- "Storage" - Supabase Storage for videos, documents, and recordings.
- "Voice" - Web Speech API for voice recognition and text-to-speech.

---

- Video lesson management with progress tracking and completion analytics.

## # SYSTEM BOUNDARIES

### - Technical Boundaries

- Single-instance deployment without load balancing.
- Concurrent user limit of 1000 plus users per instance.
- File upload limit of 50 megabytes per file.
- Database query optimization for datasets under 1 million records.
- Real-time polling with 10-second intervals for updates.

### - Functional Boundaries

- Voice commands limited to predefined command set.
- AI tutor limited to document analysis and Q&A.
- Analytics limited to historical data within 12 months.
- Messaging limited to text-based communication.
- Community features limited to posts, replies, and comments.

### - Geographic Boundaries

- Single language support for voice navigation - Hindi and English.
- Timezone support for scheduling and reporting.
- No region-specific compliance beyond WCAG accessibility.

---

## # CONSTRAINTS AND LIMITATIONS

- Backend server must be running on localhost:3001 for development.
- Ollama AI service required for tutor functionality.
- Modern browser with Web Speech API support required for voice features.
- PostgreSQL database required for data storage.
- Supabase account required for authentication and storage.
- Internet connection required for all features.
- No offline capability for content or features.
